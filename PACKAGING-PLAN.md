# Zouk Skills & Agents - Packaging Plan

This file defines the final download experience for buyers.

## Buyer Goal

When a buyer opens the zip, it should feel organized, premium, and easy to understand within the first minute.

## Final Zip Name

Use this zip name for delivery:

Zouk-Skills-Agents-v1.zip

## Final Extracted Folder Name

The zip should extract into:

Zouk-Skills-Agents/

Do not ship a folder named after the GitHub repo slug.

## Buyer-Facing Root Order

Keep these files at the top level:

1. 00-START-HERE.md
2. README.md
3. QUICKSTART.md
4. PRODUCT.md
5. FEATURES.md
6. LICENSE-COMMERCIAL.md
7. CHANGELOG.md
8. VERSION.txt
9. CORE.md
10. LOADOUTS.md
11. CATALOG.md

## Buyer-Facing Folders

Use these clean folders:

- agents/
- skills/
- workflows/
- loadouts/
- tools/
- resources/
- docs/
- bonus/

## Bonus Folder

Put buyer bonus material inside:

bonus/

Recommended bonus file:

CHEATCODE-PROMPTS.pdf

## Do Not Include In Final Zip

Remove or exclude:

- .git/
- node_modules/
- .DS_Store
- temporary files
- old backups
- duplicate drafts
- hidden local environment files
- private keys
- raw secrets
- local install state

## Product Feel Checklist

Before shipping, confirm:

- The first file is 00-START-HERE.md
- The zip name is clean
- The extracted folder name is clean
- The product name is Zouk Skills & Agents everywhere buyer-facing
- There are no ChatGPT, OpenAI, generated image, temp, draft, or backup names in buyer-facing assets
- The commercial license is included
- The cheatcode prompt bonus is included
- The README points buyers to the start file

## Final Buyer Experience

The buyer should immediately understand:

- what they bought
- how to start
- what is included
- what is bonus material
- what license they have
- how to use the system safely
